  • 语言基础: 与Java有所不同, Python不是依赖于编译器的语言, 而是与C, Ruby一样是基于解释器的语言, 也是一种动态语言.

  • 标识符:

    • 严格区分大小写
    • 下划线开头的标识符在python中有特殊含义, 一般情况下应该避免使用相似标识符

  • 标识符的命名规则

    • 模块(py文件)名称尽量短小, 并尽量使用小写字母, 可以使用下划线分隔多个字符. (与Java文件首字母大写不同不同)

    • 包名(package)尽量短小, 并尽量使用小写字母, 不推荐使用下划线分隔, 可以使用.进行分隔.

    • 类名首字母大写(同Java)

    • 模块内部类使用下划线开头, 例如_InnerMyClass这种写法

    • 函数, 类的属性和方法全部使用小写字母, 多个单词之间可以使用下划线进行分隔

    • 例如以下文件命名为my_python_main.py

  • 标识符的特殊注意事项 (暂时还没摸到实际例子)

    • 使用单下划线”_”开头的模块变量或函数是受保护的, 在使用from xxx import *语句导入的时候, 这些模块变量和函数是不能被导入的
    • 使用双下划线”__”开头的实例变量或方法是这个类私有的
    • 以双下划线”__”开头和结尾是python的专用标识, 例如__init__()代表初始化函数.

  • 变量:

    • 在定义变量时, 不需要声明变量数据类型

    • 必须是一个有效的标识符 (不能以数字和下划线开头)

    • 不能使用python中的保留字

  • 常量

    • python中没有定义常量的保留字

    • 常量命名规则与Java类似, 由大写字母和下划线组成

    • 常量赋值后, 后续仍然可以被修改. 并不存在像Java中的final关键字.

  • 数据类型

    • 整数类型

      • python中的整数类型为不可变数据类型

      • 理论上的取值范围是从[正无穷,负无穷]

      • 可以使用二进制, 八进制, 十六进制直接给变量赋值

    • 浮点数类型

      • 可以使用带e的科学计数法, 不可变数据类型

      • 必须带有小数部分, 小数部分可以为0

      • 有一定概率在计算后, 带有不确定的尾数

      • 可以使用内置函数round()限定运算结果需要保留的小数位数

    • 复数类型 ( 科学计算以外很少用到 )

      • 复数由实部和虚部组成, 不可变数据类型

      • .real获取实数部分, .imag获取虚数部分

      • j为复数的一个基本单位, 被定义为j=根下-1, 称为虚数单位

    • 字符串

      • 不可变数据类型, 即不可变字符序列

      • 单行字符串’str’或”str”

      • 多行字符串’’’str’’’或”””str”””

      • 转义符\对特殊符号进行转义

      • 在字符串界定符前加上r或R, 可以使转义符失效, 直接输出字符串. 我X这可比Java舒服太多了?

      • 字符串索引, 正向索引从0开始, 反向索引最后一位为-1

      • subString的写法: str[M:N] , 与Java相同, [M,N) 的取值范围. M不写的时候默认为0, N默认直接到结尾. 方向均为字符串从左向右.

      • 加号可以连接字符串, 但更推荐逗号连接, 因为不同数据类型会报错. 星号可以复制多次字符串. in可以判断是否在字符串中, 返回值为boolean. 也比Java舒服太多了, Java的contains只能判断char是否在String中, string要使用indexOf(str)的返回值非-1来判断是否含有字符串.

    • 布尔类型

      • 可以转换为数值True为1, False为0. 首字母大写

      • 所有对象都有一个布尔值, 使用bool()进行测试

      • 布尔值为假的情况

        • False或者None

        • 数值中的0, 包含0, 0.0, 虚数0

        • 空序列, 包含空字符串, 空元组, 空列表, 空字典 ( 咦? 这个会不会很好用? )

        • 自定义对象的实例, 该对象的__bool__()方法返回False或者__len__()方法返回0

    • 数据类型转换

      • 隐性数据类型转换: 如int类型与float类型进行运算时, 结果自动转换为float类型

      • 显性数据类型转换: 类似Java中的强转, int(变量名)这种玩法.但python的转换方法比起Java简单易用很多很多

Java基础编程思想复习2

Java基础编程思想复习2

抽象类 abstract

  • 抽象类的概念

    • 父类知道子类要完成某个功能, 但每个子类的实现是不同的.
    • 子类以后也只会用自己重写的功能, 那么父类就可以把这个功能定义成抽象方法
    • 抽象类就是拥有抽象方法的类
    • 抽象方法: 没有方法体, 只有方法签名和返回值, 必须用abstract修饰

  • 抽象类的使用

    • 抽象类的作用就是为了让子类继承, 约束子类一定要重写方法
    • 一个类继承了抽象类, 必须要实现抽象类中的所有抽象方法, 否则这个类也只能是抽象类.

  • 抽象类的特征: 有得有失

    • 有得: 抽象类得到了拥有抽象方法的能力, 也就是约束子类的能力
    • 有失: 抽象类失去了创建对象的能力 (抽象类虽然有构造器, 但不能实例化)
    • 抽象类本身就意味着抽象, 无法具体化

  • 抽象类存在的意义

    • 抽象类的存在就是为了被子类继承, 对子类进行约束, 否贼抽象类毫无意义
    • 抽象类体现的是模版思想, 部分实现, 部分抽象. 模版设计思想

  • 抽象类与设计模式

    • 设计模式是前人在开发过程中发现的优秀软件架构和思想, 后者可以直接使用, 提高效率, 提高可扩展性和可维护性.
    • 模版模式就是一种经典的设计模式
    • 作用: 优化代码架构, 提高代码复用性, 相同功能的重复代码无需重复书写. 可以部分抽象, 部分实现, 抽象的东西交给后人去实现. 例如模版作文?

  • 抽象类的注意事项

    • 抽象类不能创建对象, 如果创建, 编译无法通过, 必须通过其子类创建对象.

      理解: 假设创建了抽象类的对象, 那么调用抽象方法没有方法体, 没有意义.

    • 抽象类一定有而且必须有构造器, 是供子类创建对象时, 初始化父类对象用的.

      理解: 子类的构造器中, 有默认的super构造器, 需要访问父类构造器.

    • 抽象类中, 不一定包含抽象方法, 但包含抽象方法的一定是抽象类.

    • 抽象类的子类, 必须重写父类中的所有抽象方法, 否则子类也只能是抽象类

    • 抽象类存在的意义就是为了被子类继承, 体现了模版设计思想

      理解: 抽象类中已经实现的是模版中确定的成员

      抽象类中不确定如何实现的定义为抽象方法, 交给它的子类去实现.

接口 interface

  • 接口的概述

    • 接口是更加彻底的抽象, 在Java8之前接口中只能是抽象方法和常量, 没有其他成分 ( 构造器等 ).
      1. 抽象方法: 接口中定义的方法可以省略public和abstract不写, 默认是加上的 ( 但抽象类中不能省略, 因为抽象类中可能存在非抽象方法 )
      2. 常量: 变量值只有一个, 而且在程序执行中不能被改变, 常量修饰符一般是public static final
        常量的变量名称一般建议字母全部大写, 多个单词用下划线连接
        接口中可以省略public static final
    • 接口体现的是规范的思想, 实现接口的子类必须重写接口中的所有方法.
    • 抽象类也需要实现全部抽象方法, 但如果是子类声明为抽象类, 则不必.

  • 接口的实现

    • 接口的实现类必须实现接口中的全部抽象方法, 否则就成了用来空实现的抽象类.

    • 一个实现类可以实现多个接口, implements后面以逗号分隔, 但也需要实现全部方法.

    • 类与类是单继承关系, 类与接口是多实现关系, 接口与接口也是多实现关系

    • 实现多个接口时, 多个接口的方法名称不能重复, 否则报错.

      解决方法: 实现类中创建内部类, 分别实现两个接口

  • Java8以后的接口

    • Java8以前, 接口中只有抽象方法和常量. 8版本以后接口不再纯净, 新增了3个方法
      1. 默认方法: 其实就是实例方法, 必须用default修饰, 默认也会加上public. 只能有接口的实现类调用.
      2. 静态方法: 可以直接加static修饰, 默认也会加上public. 只能用接口名调用, 不能被实现类调用.
      3. 私有方法: 其实就是私有的实例方法, 只能在本接口内被访问, 必须加上private修饰, 从Java9开始的. 通常是用来给默认方法调用的.

  • 实现多个接口的注意事项

    • 如果实现多个接口, 多个接口中存在的同名的静态方法并不会冲突, 因为通过接口名称调用.

    • 当一个类继承了一个父类, 又实现了多个接口, 父类中的成员方法与接口中的默认方法重名时, 子类就近选择父类方法执行.

    • 当一个类实现多个接口时, 多个接口中存在重名的默认方法, 那么子类必须重写这个方法.

    • 接口中没有构造器, 不能创建对象. 接口是更彻底的抽象, 连构造器都没有, 当然不能实例化.

    • 接口: 东西少, 但是可以多实现.

      抽象类: 东西多, 但是不能多继承.

代码块

  • 类的五大成分之一: 成员变量, 成员方法, 构造器, 代码块, 内部类
  • 静态代码块
    • 必须用static修饰, 属于类, 会与类一起加载, 而且自动触发执行一次.
    • 静态代码块可以用于在执行类的方法之前进行静态资源的初始化操作
  • 实例代码块
    • 必须无修饰, 属于类的每个对象, 与类的每个对象一起加载, 每次创建对象时, 代码块就会被触发执行一次
    • 实例代码块可以用于初始化实例资源
    • 实例代码块实际上是提到构造器中去执行的.

final关键字

  • final关键字概述

    • final是终极的含义, 可以用于修饰类, 方法, 变量.

    • final修饰类: 类不能被继承, 断子绝孙了.

      理解: MIUI就是牛逼, MIUI天下第一, 别从谷歌继承, 直接用MIUI

    • final修饰方法: 方法不能被重写

    • final修饰变量: 变量有且只能被赋值一次

    • 拓展:

      • abstract和final的关系是什么? 互斥关系, 无法同时出现.
      • final修饰局部变量, 就是让值被固定或者说被保护起来, 执行过程中防止被更改.

  • final修饰变量

    • final修饰变量总规则: 有且仅能被赋值一次.
    • final修饰静态变量: 变量就成了常量.
    • 常量: 有public static final修饰, 变量名称全部大写, 多个单词之间用下划线连接.

  • 拓展

    • final修饰静态成员变量可以在哪些地方被赋值一次
      1. 定义变量的时候赋值
      2. 可以在静态代码块中赋值一次
    • 修饰实例成员变量的意义不大, 可以定义的时候赋值, 可以实例在实例代码块中赋值, 个人感觉不如构造器, 脱裤子放屁.

单例设计模式

  • 什么是单例设计模式

    单例的意思是一个类永远只存在一个对象, 不能创建多个对象

  • 为什么要使用单例

    开发中很多类的对象我们只需要一个, 如虚拟机, 任务管理器等.
    对象越多, 占用资源越多, 单例设计模式节约内存资源, 提高性能

  • 单例设计模式的实现总共有8种, 此处只讲述两种.

    • 饿汉模式

      • 通过类获取单例对象的时候, 单例已经提前做好了.

      • 实现步骤

        1. 定义一个单例类
        2. 把构造器私有, 藏起来
        3. 定义一个静态成员变量, 用于存储一个对象
          ( 饿汉单例要求在返回对象时, 对象要求已经做好, 所以这里直接创建 )
        4. 定义一个方法返回单例对象

    • 懒汉模式

      • 通过类获取对象的时候, 发现没有对象, 才去创建一个对象.

      • 实现步骤

        1. 定义一个单例类
        2. 把构造器私有, 藏起来
        3. 定义一个静态成员变量, 用于存储一个对象
          ( 懒汉单例中不创建对象 )
        4. 定义一个方法, 当3中的变量为空时, 返回单例对象.

枚举

  • 枚举类的概述

    • 作用: 用于做信息标志和信息分类

    • 格式: 修饰符 enum 枚举名称{ 实例1名称, 实例2名称 }

      第一行必须是枚举类的对象名称, 需要大写.

    • 枚举的特点

      1. 枚举类是final修饰的, 不能被继承( 可以通过javap反编译看底层代码 )
      2. 枚举类默认继承了枚举类型: java.lang.Enum
      3. 枚举类第一行罗列的是枚举类的对象, 而且是用常量储存的
      4. 所以枚举类的第一行都是常量名称, 默认存储了枚举对象
      5. 枚举类的构造器默认就是私有的
      6. 枚举相当于是多例设计模式

Java基础编程思想复习1

发表于 2021-05-12 更新于 2021-05-18 分类于 Java ， 基础 ， 编程思想

Java基础编程思想复习1

  • 知识回顾

    • 面向对象:

      在代码层面, 类中有且仅有5种成分.

      • 成员变量

      • 成员方法

      • 构造器

        ( 类中默认自带一个无参构造器, 即使不写它也是存在的, 如果它写了一个构造器, 那么无参构造器就被覆盖了)

      • 代码块

      • 内部类

      只要不是以上五种成分, 就必定报错.

      this关键字: this关键字代表当前对象的引用, 可以在构造器和实例方法中使用. this在方法中时, 谁调用这个方法, this就代表谁, 在构造器中, this代表了当前正在初始化的对象的引用.

  • static关键字

    有static修饰的变量或方法属于类本身, 变量与类一起加载一次, 可以通过类名直接调用.
    无static修饰的变量或方法属于类的每个对象, 叫做实例变量或实例方法, 有多少个对象, 变量就加载多少次, 必须通过类的对象调用.
    实例可以访问静态变量或方法, 静态可以访问静态变量或方法, 静态不可以访问实例变量或方法.

  • 继承:

    继承的作用:

    • 继承可以提高代码的复用性, 相同代码向上抽取到父类.
    • 子类直接继承父类, 可以得到父类的方法, 还可以定义子类特有方法, 通常子类更强大

    通常情况下, 若几个类有相同的功能或方法, 那么就把他们向上抽取到父类中, 重复利用.

    注意:

    • 子类不能继承父类的构造器, 子类有自己的构造器
    • 子类是否可以继承父类的私有成员变量或方法? 可以, 但无法直接访问.
    • 子类是否可以得到父类的静态成员? 不可以, 但可以共享访问.

    继承后成员变量的访问特点:

    • 就近原则: 子类有就找子类, 子类没有就找父类, 再没有就报错
    • this代表当前对象的引用, super代表了父类对象的引用

    继承后方法的重写:

    • 子类继承父类方法, 但无法满足业务需求, 需要重写方法内容进行覆盖
    • 方法重写时需要加上@Override注解: 可读性好，安全，优雅

    方法重写的要求:

    • 方法名称和形参列表必须与父类方法保持一致
    • 返回值类型要么与父类保持一致, 要么范围比父类更小
    • 权限修饰符应该与父类相同或者比父类更大
    • 抛出的异常应该与父类一致或范围更小
    • 总体建议: 加上@Override注解, “申明不变, 重新实现”, 私有方法不能被重写.

    继承后构造器的特点:

    • 子类的构造器第一行默认访问父类的无参构造器, 再执行自己的构造器

    • 为什么:

      • 子类构造器默认第一行为super(); 调用父类午餐构造器, 写不写都存在

        ( 如果父类中没有无参构造器, 编译也就会失败. )

      • 子类继承父类就得到了父类的属性和行为. 调用子类构造器初始化子类对象的时候, 必须先调用父类构造器初始化父类的属性和行为. 正所谓先有爹后有儿.

    继承后super调用父类构造器:

    • 具体场景: 可以在子类构造器中通过super(…);依据参数调用父类构造器, 以便调用父类构造器初始化继承自父类的数据.

    • class Monkey extends Animal{
    public Monkey(String name , int age , char sex) {
        super(name, age, sex);
    }

    public void eatBanana(){
        System.out.println(getName() + "-->" + getAge() + "-->" + getSex() + "在吃香蕉");
    }
}

  • this和super关键字的使用:

    • this(…): 可以用来接用本类其他构造器
    • super(…): 调用父类构造器

  • 继承的特点总结:

    • 单继承：一个类只能继承一个直接父类。

    • 为什么Java是单继承？

      答：反证法，假如C继承A和B，若AB中有同名方法，C调用就懵逼了

    • 多层继承：一个类可以间接继承多个父类。（家谱）

    • 一个类可以有多个子类

    • 一个类要么默认继承了Object类，要么间接继承了Object类，祖宗类。
